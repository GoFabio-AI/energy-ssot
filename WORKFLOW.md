# Workflow SSOT Energia

## Obiettivo
Mantenere una base di conoscenza normativa rigorosa, verificabile e utile per decisioni operative.

## Flusso corretto
1. Fonte ufficiale
2. Registrazione in `SOURCE-REGISTRY.md`
3. Estrazione del fatto normativo
4. Creazione o aggiornamento della regola atomica in `RULES/`
5. Aggiornamento del file dominio in `DOMAINS/`
6. Solo dopo, eventuale decisione applicativa in `DECISIONS/`

## Distinzioni obbligatorie
- Fonte = documento ufficiale
- Fatto normativo = cosa dice davvero la fonte
- Interpretazione minima = chiarimento strettamente necessario
- Decisione prodotto = scelta di SUPER basata sul fatto normativo

## Divieti
- Non scrivere nel SSOT cose senza fonte
- Non mescolare interpretazione e norma
- Non usare fonti secondarie come base primaria
- Non promuovere a memoria lunga dettagli non consolidati

## Standard qualità
Ogni regola deve indicare:
- fonte primaria
- citazione precisa
- stato di vigenza
- impatto operativo
- eventuali conflitti o limiti
